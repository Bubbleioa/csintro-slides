数字的表示

为什么只能有 0 和 1？试试[三进制计算机](https://en.wikipedia.org/wiki/Ternary_computer)！ - 点人数数手指头，一进制，表示自然数 - 使用 0～9，十进制 - 使用 0, 1，二进制 每个二进制数字也被称为一个比特。 大多数计算机每次使用 8 位，称为一个字节，如 **0000000011** 代表数字 3。

无脑答：0和1 算是一个正确的答案，奖励你这个键盘，用它来构造整个计算机世界吧！

程序是如何表示的？(cont'd)

实际上远比大家想的简单。 我们可以人为约定，一个10个字节的程序其中一半是数据区域，一半是逻辑区域，我们运行一个程序，就是从逻辑区域每次取一个字节，然后根据下面这个表做出相应的动作： | 操作类型 | 数字一 |数字二| | ---- | ---- |----| | xxxx | xx |xx|

程序是如何表示的？(cont'd)

| 操作类型 | 数字一 |数字二|说明| | ---- | ---- |----|----| | 0001 | xx |xx|数字一加上数字二存放在数字一里| | 0002 | xx |xx|减法同上| | 0003 | xx |xx|将数字一复制一份到数字二| | 0004 | xx |xx|将数字一赋值为数字二| 这个表其实是我们的 CPU “提供”的，被称为指令集架构(ISA, Instruction Set Architecture) 顺便回答了之前的问题： 最小的计算机就是上面的表（CPU）加上存放程序的元件（内存）构成的。

套娃🪆的开始：汇编

总不可能真的用上面那款键盘编程吧（除非不是碳基生物了） 于是引入记号便顺理成章了： | 助记符 | 数字一 |数字二|说明| | ---- | ---- |----|----| | ADD | xx |xx|数字一加上数字二存放在数字一里| | SUB | xx |xx|减法同上| | MOV | xx |xx|将数字一复制一份到数字二| | MOVI | xx |xx|将数字一赋值为数字二| 事实上，你们在大二的时候真的要学一门汇编——[8086](https://yjdoc2.github.io/8086-emulator-web/compile)

继续套娃🪆

汇编也有很多问题，比如说比较费程序员，换一个机器就要重新写代码。 于是我们的前辈们就想：能不能开发种语言，将这种语言写的代码通过一段程序变成汇编，然后就能用前辈的前辈们实现的汇编器，变成能被计算机识别的01序列了！ C 语言就是那个时代的一员。

穿越时空的 C 语言

发明 C 语言的人肯定是最懂 C 语言的人之一，假设我们穿越回1972年 Dennis Ritchie 正好在开设补习班教大家 C 语言，除了语言不同之外，在那里学 C 语言和我们现在学 C 语言有什么区别呢？

CLI(command-line interface)

1. 使用 CLI 来操作计算机本身就是编程。 2. 能够很方便与现有工具结合使用。 3. 如果你知道命令， CLI 可以比任何其他类型的界面快得多，效率也高。它还可以轻松处理重复性任务。 4. 世界上很多计算机是没有图形界面的，而大多数人工作后都要接触它们。 5. CLI 能让你学到更多。
